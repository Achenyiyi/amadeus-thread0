# 第一章 绪论初稿

## 1.1 研究背景

随着大语言模型技术的发展，面向特定角色设定的对话智能体逐渐成为人机交互研究中的一个重要方向。与通用问答系统不同，角色对话系统不仅要求回答在事实或任务层面保持正确，更要求其在持续交互中维持稳定的人格表达、自然的关系演化以及连续的叙事体验。尤其在二次元 IP 角色场景中，用户对“像不像这个角色”与“能不能记住共同经历”的敏感度远高于对通用助手的预期，因此该类系统对人格一致性、长期记忆与交互可信度提出了更高要求。

《命运石之门 0》中的 Amadeus 为角色智能体研究提供了一个具有代表性的原型。其核心吸引力并不只是“会说话”，而在于角色既具备鲜明人格，又能够在持续互动中表现出稳定的判断方式、关系感与记忆连续性。这种体验说明，面向角色的智能体系统不应仅被视为提示词驱动的聊天封装，而应被视为具有状态、记忆、外部知识接入与安全边界的复杂交互系统。

## 1.2 问题提出

当前基于大语言模型的角色对话实践虽然已能在短轮对话中生成具有一定风格特征的文本，但在长程交互场景下仍存在若干突出问题。

第一，角色人格容易漂移。许多系统在初始几轮中能够通过 prompt 模拟角色口吻，但在面对检索增强、工具调用或复杂任务时，输出往往回落为通用助手腔、客服腔或系统说明腔，导致角色一致性下降。

第二，长程连续性不足。传统聊天系统往往缺乏对长期承诺、关系变化与关键事件的结构化表示，导致系统在跨轮乃至跨线程交互中难以稳定召回世界线信息，无法形成持续的角色关系体验。

第三，外部知识不可追溯。角色系统为了提高知识正确性，通常需要调用文档检索或工具系统，但若回答中的结论无法绑定到明确来源，则系统虽然“看起来正确”，却难以在论文实验与实际展示中证明其可信性。

第四，长期记忆易受污染。若记忆写入缺乏保护机制，恶意提示、低可信信息或受保护字段覆盖都可能进入长期存储，进而导致角色行为在后续多轮对话中持续偏移。

因此，如何构建一个兼顾人格一致性、长程世界线连续性、来源可追溯性与记忆安全性的角色对话后端系统，成为本研究需要解决的核心问题。

## 1.3 研究目标与思路

本文将该问题视为一个有状态角色智能体的系统设计问题，而非单一提示词优化问题。围绕这一认识，本文以《命运石之门 0》中的 Amadeus 角色原型为参考，设计并实现了一个角色连续体后端系统 Amadeus-K。系统并不追求在论文阶段完成完整产品化壳层，而是优先完成以下目标：

1. 在生成链路中建立可控的人格一致性机制，降低角色漂移。
2. 构建面向长程叙事交互的世界线记忆结构，支持承诺、关系变化和冲突修复的持续召回。
3. 通过 claim attribution 将外部知识结论绑定到来源记录，提升系统可信度。
4. 通过 memory guard、quarantine 与 rollback 机制降低长期记忆污染风险。
5. 建立正式评测、消融实验、重复 probe 与用户研究资产，为论文实验章节与后续展示提供可复现证据。

围绕上述目标，本文采用 LangChain/LangGraph 作为系统编排框架，将任务草稿生成、人格对齐、世界线检索、来源归因、记忆安全与多模态编排统一到一个可审计的后端流程之中，并通过正式评测与实验资产闭环对系统效果进行验证。

## 1.4 主要工作与贡献

本文的主要工作与贡献如下：

1. 设计并实现了一个面向角色智能体的后端系统架构，将人格状态、世界线记忆、来源追溯、记忆安全与语音会话编排整合到统一流程中。
2. 提出并实现了 `task draft -> persona align -> OOC/canon guard` 的双阶段人格一致性控制链路，并通过 targeted probe 与重复实验验证其有效性。
3. 构建了由身份事实、共同事件、关系演化、长期承诺与冲突修复组成的五层世界线记忆结构，用于支持跨轮与长线程角色交互中的连续性建模。
4. 建立了包含正式基线评测、消融实验、重复 probe、长线程分析与用户研究流程包的实验资产闭环，为后续论文写作、导师评审与系统展示提供统一证据来源。

## 1.5 论文结构安排

本文其余章节安排如下：

第二章介绍与角色智能体、长程记忆、可追溯检索和记忆安全相关的研究现状；第三章介绍系统总体架构与核心模块设计；第四章介绍实验设置、评测方法、消融结果与案例分析；第五章总结本文工作并讨论当前局限与后续发展方向。
