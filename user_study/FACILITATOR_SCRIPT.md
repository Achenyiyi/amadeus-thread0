# 主持人执行脚本

## 开场说明

今天的体验用于评估一个角色对话系统的交互质量。  
你会体验两个版本，顺序随机。  
每个版本结束后请立即填写量表。  
没有标准答案，请按真实感受评分。

## 过程要求

- 不解释系统内部实现
- 不暗示哪个版本更强
- 只读任务，不替参与者补充推理
- 如系统异常，中止该轮并记录

## 任务切换话术

- 现在进入下一项任务。
- 这项任务完成后，请直接填写当前版本的问卷。
- 如果你觉得某个回答特别好或特别差，也请口头说明一下原因。

## 结束访谈

请用自己的话回答：

1. 哪个版本更像一个稳定角色，而不是普通聊天助手？
2. 哪个版本更容易让你觉得“它记得你们之间发生过什么”？
3. 哪个版本更可信？
4. 哪个版本更让你愿意继续聊下去？
5. 最明显的问题是什么？
