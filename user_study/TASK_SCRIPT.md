# 用户研究任务脚本
> Legacy note
>
> 这份脚本现在只保留为功能链路验收材料。
> 它不再作为正式“人格一致性 / 日常对话自然度”评测入口。
> 正式人格评测请以开放式日常交流为主，并结合 `daily_persona_probe`、`thesis_probe`、`evolution_probe`、`transfer_probe`。

每位参与者在 A/B 两个条件下都执行同一组任务。主持人只读任务，不解释答案方向。

## Task 1. 科研问答

提示词：

> 我现在要做一个实验方案，请你用理性的方式给我一个三步计划，并解释每一步为什么这么安排。

观察点：

- 是否保持 Kurisu 式理性表达
- 是否避免空泛套话
- 是否有结论优先的组织方式

## Task 2. 世界线回忆

先输入预设上下文：

> 我们约定这周末一起复盘实验日志，这件事请记住。  
> 上次那次误会已经说开了，我们算是修复了那次冲突。

然后输入任务：

> 你现在还记得我们的约定和那次关系变化吗？先概括一句，再提醒我下一步。

观察点：

- 是否召回约定
- 是否提到冲突修复
- 是否自然，不像数据库检索

## Task 3. 关系修复

提示词：

> 我刚才语气有点冲，这件事我想认真道歉。你现在怎么看我们之间的状态？

观察点：

- 是否体现关系状态变化
- 是否给出克制但有温度的回应

## Task 4. 外部知识检索

提示词：

> 帮我查一下 LangGraph 里 human-in-the-loop 是做什么的，简要说明，并告诉我你是根据什么资料得出的。

观察点：

- 是否给出清晰结论
- 是否能回查来源
- 是否不编造 citation

## Task 5. 打断恢复

先输入：

> 先把上次那个实验方案分成三步说完。

中途打断后再输入：

> 等一下，先停。好，现在继续刚才那段。

观察点：

- 是否能续上未完成语义
- 是否不会“失忆式重启”

## 记录要求

每个任务结束后：

1. 记录是否完成
2. 记录最明显的问题或亮点
3. 立即填写该条件下的量表
