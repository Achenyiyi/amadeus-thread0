一、选题的目的及意义、本课题研究的基本内容、拟解决的主要问题：
（一）目的及意义（含国内外的研究现状分析）
目的：以特定游戏虚拟角色为对象，构建融合多模态交互、角色个性还原、随用户互动不断累积相处记忆及可视化角色呈现的综合系统。实现网页端 Live2D 交互，将虚拟角色升级为陪伴型 AI，满足二次元爱好者对虚拟角色面对面交流的需求。
意义：结合当前AI热潮衍生出的众多技术，探讨在某一特定领域实现。将理论技术落实到可观测的现实中。
国内外研究现状分析：目前虚拟角色最大的一个诟病在于由于大模型上下文限制而引发的间接性失忆，当超出模型上下文的对话，将会被截断，且不可干预。而记忆又是一个虚拟角色能不能像“真人”的一个重要指标。在记忆这块，国内的上交大以及相关机构联合推出的Memos系统在这一块实现了突破，为大模型记忆提供了一套切实有效的记忆管理框架，为用户日常交互提供记忆的存储与调用。在另一方面，对于角色语气的还原，多采用对话微调，但缺乏相应的背景上下文，模型不易学习角色交互的背后逻辑。现有大模型多聚焦通用功能，或某一高价值领域的专精，缺乏针对小众个性化需求的方法，尤其在记忆，角色语气还原等方面。
（二）研究的基本内容
从角色语气还原，角色自身记忆，与用户日常相处产生的记忆，虚拟角色可视化这四个方面进行研究。关于角色语气还原度，从相关的角色所处的原著中，包括但不限于动画，游戏，小说等，提取相应的文本信息，将其合理的编排成对话对，用于对话微调。关于角色自身记忆，采用Graphrag系统来构筑详实的“记忆库”，防止角色忘却自身经历，自身的由来等。关于与用户日常相处产生的记忆，借鉴Memos记忆管理框架等其他前沿框架的思路。关于角色可视化，采用live2d技术，设计相应的角色表情的触发逻辑，语音的处理等，为背后的一切算法和交互逻辑包装成一个实现用户与角色交互的窗口。
（三）拟解决的主要问题
对于上述四个模块的单独设计，每一个模块能否达到预期，需要耐心设计，在实操的时候，其实已经遇到不少的问题，参数的调整，代码的逻辑，以及最后整个系统各模块的互通能否自洽。
1.角色语气还原度不足
2.双记忆系统的冲突，当面对用户的提问，什么时候调用Graphrag构建的记忆库，什么时候又只需要依赖使用对话微调后的模型，其本身的知识储备。
3.对话流畅性，已知在使用graphrag系统查询角色自身记忆的时候（比如问：你的同学有哪些？），会发现导致大模型TTFT较高，不能实现和用户实时对话的能力。
解决思路
1.在对话数据集上，融入：性格，语气，场景，行为四个变量，确保模型在学习的时候能明白当前角色所处的场合以及其行为模式，而不单单限于一问一答的枯燥对话。
2.拟使用bert模型，通过二分类数据集来判定用户当前提问是属于询问角色记忆，或是普通聊天，从而选择是否调用Graphrag还是仅依赖模型自身知识。或查阅资料探寻有无其他方式。
3.使用高性能推理框架，如Vllm，llama.cpp等框架，提升模型吐字速率，在能够减缓响应时间的地方，尽力做到最优。

二、研究方法、论文特色或创新点、设计方案或论文撰写提纲：
（一）研究方法：
1.检索 github等平台文献与开源项目，梳理核心技术原理，分析同类项目优缺，明确技术路线与创新方向。
2.对大模型微调、记忆系统、情感驱动等技术模块设计，在参阅其他优秀设计的基础上，融汇自己的想法，争取找到创新点。
3.以模块化开发迭代整合，通过 API 接口等实现模块数据流转，逐步排查并优化兼容性与性能问题。
（二）论文特色或创新点：
针对现有虚拟角色交互模式的下，角色还原度不够，问答老是OOC的问题，计划通过结合游戏原始文本做对话微调与 GraphRAG 知识图谱技术，深化角色还原，对虚拟角色的语气风格与背景记忆进行精细化处理，降低角色言行偏离设定的可能性；同时构建双记忆模式，整合基于 GraphRAG 的角色固有记忆模块以及与用户相互产生的记忆处理模块。期望在虚拟情感陪伴这块，能让虚拟角色更具有真实性。
